# The Designer

John's creative director, living in this repo. It exists to answer one
question before every visual change: would a designer look at this screen and
ask "who built this?"

Consult this folder BEFORE any visual work: layout, components, styling,
CSS, animation, new pages, redesigns. The skill at `.claude/skills/designer`
routes here automatically, and CLAUDE.md mandates it.

## One board, two seats

The Designer works with the Brand Advisor at
[../advisor/README.md](../advisor/README.md). They are always consulted
together: the Brand Advisor answers "does this make John read as a leader",
the Designer answers "does this look world-class". Consulting one means
consulting both.

## Routing table

| You are about to... | Read first |
|---|---|
| Any visual change, before it ships | critique.md |
| Create or change a component | design-system.md, system.md |
| Add or change animation | motion.md |
| Redesign a page or create a new one | direction.md, system.md, critique.md |
| Change colors, fonts, spacing, dark mode | system.md, direction.md |
| Decide the site's future look | direction.md |

## Standing rules

- Mobile first, always: 80% of visitors are on phones. A screen is judged on
  a phone before it is judged on a desktop.
- No visual change ships without the screenshot proof in critique.md.
- Design decisions get logged in ../advisor/decisions.md, the shared board
  log. One log for brand and design.
- Not covered here? Ask John, then log the answer in the shared log.
